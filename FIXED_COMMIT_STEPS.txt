WHY THIS HAPPENED:
══════════════════════════════════════════════════════════════════

The .tbz2 files were committed to Git before .gitignore was updated.
Git doesn't automatically "forget" tracked files when you add them
to .gitignore - you must explicitly remove them with:

  git rm --cached <file>

This removes them from Git tracking but keeps them on your disk.
